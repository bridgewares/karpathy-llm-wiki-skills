# 这个 vault 的已有规则

## 写作与维护偏好
- 保留我已有的写作习惯
- 仅补充缺失的 wiki 规则，不要整篇重写
- 现有页面标题优先使用我已经采用的命名方式
- 不要因为初始化而替换整份规则文件

## 协作边界
- 需要新增规则时，请以增量方式补充
- 如果某条默认规则与本 vault 现有习惯冲突，优先保留本 vault 的现有约束

## 个人 LLM Wiki 最小维护规则
- 这是一个由 LLM 协助维护的个人 wiki，`raw/` 保存原始资料，`wiki/` 保存整理后的知识页。
- `raw/inbox/` 用于放新来源入口，`raw/library/` 用于保存正式纳入管理的原始资料。
- `raw-index.md` 是 raw 来源 ingest 状态的唯一官方状态表；当前阶段暂不使用单文件 metadata 作为官方状态来源。
- ingest 时先把材料放入 `raw/inbox/<type>/`，确认后再整理到 `raw/library/<type>/`，随后更新 `raw-index.md` 与对应来源页。
- query 时优先查看 `index.md` 与相关 `wiki/` 页面，只有在 wiki 信息不足时再回到 `raw/`。
- 新建页面以最小可用为原则，优先更新已有页面，不为了初始化或链接而批量创建空页面。
- 每次 ingest、query、lint 或重要结构调整后都要更新 `index.md` 与 `log.md` 中对应的导航或记录。
- 文档主体优先中文；所有用于状态判断的时间字段统一使用 `YYYY-MM-DD HH:MM:SS`，精确到秒。
