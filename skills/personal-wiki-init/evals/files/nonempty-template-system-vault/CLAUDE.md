# 个人 LLM Wiki 维护规则

## 1. 目标
这是一个由 LLM 协助维护的个人 wiki，用于持续积累来源、问题与知识页，而不是每次都从原始资料重新开始。

## 2. 目录边界
- `raw/inbox/`：新来源入口，所有新资料先进入这里，按类型放到对应子目录。
- `raw/library/`：正式原始资料库，已确认并归类的 raw 来源放在这里。
- `raw-index.md`：raw 来源的唯一官方 ingest 状态表，只登记 `raw/library/` 中的来源。
- `wiki/`：整理后的知识页层，与 raw 分离。
- `templates/`：页面模板。
- `system/`：补充规则文件。

## 3. 新来源 ingest 工作流
1. 所有新来源先进入 `raw/inbox/<type>/`。
2. 确认来源类型后，将来源移动或整理到 `raw/library/<type>/`。
3. 在 `raw-index.md` 中为该来源建立或更新条目。
4. 在 `wiki/sources/` 创建或更新来源页。
5. 提取主题、概念、人物和问题，并更新相关 `wiki/` 页面。
6. 每次 ingest 后更新 `index.md` 与 `log.md`。

## 4. 查询与回答工作流
1. 先查看 `index.md`。
2. 优先读取相关 `wiki/` 页面。
3. 只有在 wiki 信息不足时再回到 `raw/`。
4. 若形成长期有价值的总结，应回写到 `wiki/questions/` 或 `wiki/syntheses/`。

## 5. 页面创建与更新原则
- 优先更新已有页面，而不是重复建页。
- 只有值得长期追踪的主题、人物、概念才单独建页。
- 信息不足时先在来源页记录，不强行拆页。
- 避免一次性生成大量空页面。

## 6. 索引与日志维护
- `index.md` 提供高价值导航，不追求穷举全部文件。
- 每次 ingest、query、lint、初始化补齐后都要在 `log.md` 追加记录。

## 7. 写作与时间格式
- 主要使用中文，保留必要英文术语。
- 不把猜测写成事实，冲突信息需明确标注。
- 所有用于状态判断的时间字段必须精确到秒，统一使用 `YYYY-MM-DD HH:MM:SS`。
