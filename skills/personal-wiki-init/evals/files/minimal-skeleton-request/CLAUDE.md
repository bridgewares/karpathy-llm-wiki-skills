# 个人 LLM Wiki 维护规则

## 1. 目标
这是一个由 LLM 协助维护的个人 wiki。目标是先建立最小可用骨架，清楚区分原始资料层与知识页层，并支持后续持续 ingest。

## 2. 目录边界
- `raw/`：原始资料层，不直接承载知识整理。
- `raw/inbox/`：新来源入口，所有新资料先放这里等待处理。
- `raw/library/`：正式原始资料库，仅保存已确认纳入管理的来源。
- `raw-index.md`：raw 来源 ingest 状态的唯一官方状态表，只登记 `raw/library/` 中的来源。
- `wiki/`：知识页层，用于来源页、主题页、概念页、问题页等整理结果。

## 3. 最小工作流
### ingest
1. 新来源先进入 `raw/inbox/` 对应类型目录。
2. 确认值得纳入后，移动到 `raw/library/` 对应类型目录。
3. 在 `raw-index.md` 登记来源与状态。
4. 在 `wiki/` 中创建或更新对应知识页。

### query
1. 先查看 `index.md` 与相关 `wiki/` 页面。
2. 只有在 wiki 信息不足时再回到 `raw/` 查看原始资料。

## 4. 页面创建与更新原则
- 优先更新已有页面，不重复建页。
- 只创建最小必要页面，不一次性铺满空页面。
- 不把猜测写成事实，结论尽量附证据来源。

## 5. 根文件维护要求
- 每次 ingest 后更新 `index.md`，保持高价值导航。
- 每次 ingest、query、lint 或重要初始化后在 `log.md` 追加记录。
- `inbox.md` 用于集中记录待处理来源、问题和动作。

## 6. 写作与时间格式
- 主体内容优先使用中文，保留必要英文术语。
- 所有用于状态判断的时间字段必须精确到秒，统一使用 `YYYY-MM-DD HH:MM:SS`。
